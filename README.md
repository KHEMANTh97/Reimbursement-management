# Institutional Ledger: Reimbursement Management System

## Overview
Companies often struggle with manual expense reimbursement processes that are time-consuming, error-prone, and lack transparency. Currently, there is no simple way to define approval flows based on thresholds, manage multi-level approvals, or support flexible approval rules. Institutional Ledger solves these challenges by automating the reimbursement workflow.

## Core Features

### Authentication & User Management
* On first login/signup, a new Company and Admin User are auto-created.
* The company's currency gets set automatically based on the environment's selected country.
* The Admin can create Employees and Managers.
* The Admin can assign and change roles (Employee, Manager).
* The Admin can define manager relationships for employees.

### Expense Submission (Employee Role)
* Employees can submit expense claims containing the Amount, Category, Description, and Date.
* The submitted amount can be in a different currency from the company's default currency.
* Employees can view their expense history, including Approved and Rejected statuses.

### OCR for Receipts (Auto-Read)
* The system includes OCR for receipts (auto-read).
* Employees can just scan a receipt and using an OCR algorithm the expense gets autogenerated.
* All necessary fields like amount, date, description, expense lines, expense type, and the name of the restaurant get filled automatically.

### Approval Workflow
* The expense is first approved by the employee's manager, if the "IS MANAGER APPROVER" field is checked.
* When multiple approvers are assigned, the Admin can define their sequence. 
* Examples of sequential flow include Step 1: Manager, Step 2: Finance, and Step 3: Director.
* The expense moves to the next approver (generating an approval request in their account) only after the current one approves or rejects it.
* Managers can view expenses waiting for approval.
* Managers can Approve/Reject claims with comments.

### Conditional Approval Flow
* Percentage rule: If a defined percentage (e.g., 60%) of approvers approve, the expense is approved.
* Specific approver rule: If a specific designated user (e.g., the CFO) approves, the expense is auto-approved.
* Hybrid rule: Combines both rules (e.g., 60% OR CFO approves).
* The system supports a combination of both Multiple approvers and Conditional flows together.

## Roles & Permissions

* Admin: Create company (auto on signup), manage users, set roles, configure approval rules, view all expenses, override approvals.
* Manager: Approve/reject expenses (amount visible in company's default currency), view team expenses, escalate as per rules.
* Employee: Submit expenses, view their own expenses, check approval status.
