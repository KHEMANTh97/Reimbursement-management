# Institutional Ledger: Reimbursement Management System

## Overview
Companies often struggle with manual expense reimbursement processes that are time-consuming, error-prone, and lack transparency[cite: 3]. Currently, there is no simple way to define approval flows based on thresholds, manage multi-level approvals, or support flexible approval rules[cite: 4, 5, 6, 7]. Institutional Ledger solves these challenges by automating the reimbursement workflow[cite: 2].

## Core Features

### Authentication & User Management
* On first login/signup, a new Company and Admin User are auto-created[cite: 10, 11].
* The company's currency gets set automatically based on the environment's selected country[cite: 11].
* The Admin can create Employees and Managers[cite: 12, 13].
* The Admin can assign and change roles (Employee, Manager)[cite: 14, 15].
* The Admin can define manager relationships for employees[cite: 16].

### Expense Submission (Employee Role)
* Employees can submit expense claims containing the Amount, Category, Description, and Date[cite: 19, 20].
* The submitted amount can be in a different currency from the company's default currency[cite: 19].
* Employees can view their expense history, including Approved and Rejected statuses[cite: 21].

### OCR for Receipts (Auto-Read)
* The system includes OCR for receipts (auto-read)[cite: 53].
* Employees can just scan a receipt and using an OCR algorithm the expense gets autogenerated[cite: 54].
* All necessary fields like amount, date, description, expense lines, expense type, and the name of the restaurant get filled automatically[cite: 54].

### Approval Workflow
* The expense is first approved by the employee's manager, if the "IS MANAGER APPROVER" field is checked[cite: 23].
* When multiple approvers are assigned, the Admin can define their sequence[cite: 24]. 
* Examples of sequential flow include Step 1: Manager, Step 2: Finance, and Step 3: Director[cite: 26, 27, 28, 29, 31, 32].
* The expense moves to the next approver (generating an approval request in their account) only after the current one approves or rejects it[cite: 33].
* Managers can view expenses waiting for approval[cite: 34, 35].
* Managers can Approve/Reject claims with comments[cite: 36].

### Conditional Approval Flow
* Percentage rule: If a defined percentage (e.g., 60%) of approvers approve, the expense is approved[cite: 40].
* Specific approver rule: If a specific designated user (e.g., the CFO) approves, the expense is auto-approved[cite: 41].
* Hybrid rule: Combines both rules (e.g., 60% OR CFO approves)[cite: 42].
* The system supports a combination of both Multiple approvers and Conditional flows together[cite: 43].

## Roles & Permissions

* Admin: Create company (auto on signup), manage users, set roles, configure approval rules, view all expenses, override approvals[cite: 47, 48].
* Manager: Approve/reject expenses (amount visible in company's default currency), view team expenses, escalate as per rules[cite: 49, 50].
* Employee: Submit expenses, view their own expenses, check approval status[cite: 51].
